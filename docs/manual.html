NapCat QQ Manager

一款基于 FastAPI + React 构建的 NapCat QQ Bot Docker 容器管理面板，提供可视化的容器生命周期管理、多节点集群控制、用户权限分配与实例级文件管理功能。

✨ 核心特性

  • 容器管理 — 一键创建/启动/停止/删除 NapCat Docker 容器
  • QR 码登录 — 实时获取容器内 QQ 登录二维码
  • 多节点集群 — 管理分布在不同服务器上的 NapCat 实例
  • 用户权限 — 管理员/普通用户分级，实例级授权
  • 文件管理 — 在线编辑容器配置文件和插件
  • 网络配置 — 可视化管理 OneBot 11 网络端点
  • 告警监控 — 基于 Webhook 的容器状态告警通知
  • 备份恢复 — 数据库一键备份与恢复
  • 定时任务 — 容器自动重启等定时操作
  • 操作日志 — 全链路操作审计记录
  • 安全防护 — CSRF/SSRF 防护、IP 封禁、随机初始密码
  • 中英双语 — 完整的国际化支持

层级技术说明

安装部署

环境要求

组件版本要求用途
Python≥ 3.10后端运行时
Node.js≥ 16前端构建（仅部署时需要）
Docker≥ 20.10容器管理功能
操作系统Windows / Linux / macOS均支持

方式一：Docker Compose 部署（推荐）

# 获取源码
git clone https://github.com/your-repo/ncqq-manager.git
cd ncqq-manager

# 一键启动（自动构建镜像并运行）
docker compose up -d
Docker Compose 会自动构建前端、安装依赖并启动服务。数据持久化在 config/ 和 data/ 目录。镜像内置 HEALTHCHECK 自动检测服务健康状态。

方式二：手动安装

1. 获取源码

git clone https://github.com/your-repo/ncqq-manager.git
cd ncqq-manager

2. 安装 Python 依赖

# 推荐使用虚拟环境
python -m venv .venv
# Windows:
.venv\Scripts\activate
# Linux/macOS:
source .venv/bin/activate

pip install -r requirements.txt

3. 构建前端

cd frontend
npm install
npm run build
cd ..
构建完成后会生成 frontend/dist/ 目录，后端自动挂载为静态资源。

4. 启动服务

# 方式一：使用快速启动脚本
python start.py

# 方式二：直接启动
python main.py

# 方式三：使用 uvicorn
uvicorn main:app --host 0.0.0.0 --port 8000

5. 访问面板

浏览器打开 http://localhost:8000，首次访问将引导您设置管理员账号。

请确保 Docker 服务已启动，否则容器管理功能将不可用。

快速启动脚本

项目提供了 start.py 一键部署脚本，自动完成环境检查、依赖安装、前端构建和服务启动。

基本用法

# 默认启动（端口 8000）
python start.py

# 指定端口
python start.py --port 9000

# 跳过前端构建（已构建过）
python start.py --skip-build

# 开发模式（代码热重载）
python start.py --dev

脚本执行流程

  1. 检查 Python 版本 — 确认 ≥ 3.10
  2. 安装 Python 依赖 — 自动 pip install -r requirements.txt
  3. 检查 Docker — 验证 Docker 服务是否运行
  4. 检查 Node.js — 验证 npm 是否可用
  5. 构建前端 — 自动 npm install && npm run build
  6. 启动服务 — 以 Uvicorn 运行 FastAPI 应用
如果前端已构建过（frontend/dist/ 存在），脚本会自动跳过构建步骤。使用 --dev 可强制重新构建。

管理面板

管理面板（Dashboard）是管理员的主操作界面，提供容器概览、批量操作和系统监控功能。

登录

首次访问会自动跳转到初始化向导（/setup），引导您设置管理员账号和密码（最少 6 个字符）。设置完成后使用该账号登录管理面板。

如果系统在没有任何用户的情况下启动，会自动生成一个随机密码的应急管理员账号，密码仅在终端日志中输出一次。请立即登录后修改密码。

面板功能

容器概览

  • 查看所有 NapCat 容器的运行状态（运行中/已停止/异常）
  • 显示每个容器的 QQ 号（UIN）、CPU/内存使用率
  • 支持按状态、名称筛选容器

快速操作

  • 创建容器 — 输入容器名称和 QQ 账号，一键创建并启动 NapCat 实例
  • 批量管理 — 启动/停止/重启/删除容器，可选删除数据
  • QR 码登录 — 实时获取容器内的 QQ 登录二维码

系统监控

  • CPU / 内存 10 分钟平均使用率（Daemon Monitor 自动采集）
  • 容器数量统计与节点健康状态

集群配置

节点管理

NapCat Manager 支持多节点集群部署，可从一个面板管理多台服务器上的容器实例。

本地节点

安装后自动创建 local 节点，指向本机 Docker 引擎。

添加远程节点

  1. 在远程服务器安装 NapCat Manager
  2. 记录远程服务的 地址:端口 和集群设置页面中的 api_key
  3. 在本机管理面板 → 节点管理 → 添加节点，填入地址和 API Key

密钥体系

存储位置字段作用
SQLite settings 表api_key本机集群通信密钥 — 其他节点访问本机时使用此 Key 认证（可在集群设置页面修改）
SQLite nodes 表api_key节点代理凭证 — 本机访问远程节点时携带对方的 Key
SQLite users 表apiKey用户 API Key — 用于免登录 API 调用认证
远程节点的 api_key 必须与对方节点集群设置中的 api_key 一致，否则代理请求会被拒绝（401）。

告警设置

告警功能可在容器状态异常时通过 Webhook 推送通知到外部服务（如企业微信、钉钉、飞书等）。

创建告警规则

  1. 进入管理面板 → 告警设置
  2. 点击"添加规则"，配置规则名称和 Webhook URL
  3. 保存后系统会在触发条件满足时自动推送通知

告警类型

类型说明
container_stop容器意外停止时触发
high_cpuCPU 使用率超过阈值时触发
high_mem内存使用率超过阈值时触发
login_failureQQ 登录失败时触发

允许所有 IP 访问

默认情况下，系统会解析 Webhook URL 的目标地址并拒绝内网/回环/保留地址（SSRF 防护）。如果通知插件与管理器部署在同一台机器上，可开启页面顶部的「允许所有 IP 访问」开关：

  • 关闭（默认）— Webhook 不可指向 127.0.0.1、10.*、172.16-31.*、192.168.* 等内网地址
  • 开启 — 允许指向任意地址，包括 localhost 和内网 IP
Webhook URL 格式兼容 AstrBot Webhook Push 插件，示例：http://<IP>:60071/common-webhook

备份恢复

备份功能允许管理员导出和恢复系统数据库，包括所有用户、配置和操作日志。

操作说明

功能说明
下载备份导出当前数据库为 .db 文件，自动创建安全副本避免锁冲突
上传恢复上传之前导出的 .db 文件覆盖当前数据库（自动备份旧数据库），需要重启生效

上传的备份文件最大不超过 50 MB，且必须为 .db 格式。恢复操作会覆盖当前数据库，请确认后再操作。

定时任务

定时任务功能允许管理员设置容器的自动化操作，如定时重启、定时启动/停止等。

创建定时任务

  1. 进入管理面板 → 定时任务
  2. 点击"添加任务"，选择目标容器和操作类型
  3. 设置执行时间（Cron 表达式或预设选项）
  4. 保存后任务将按计划自动执行

支持的操作

  • restart — 定时重启容器
  • start — 定时启动容器
  • stop — 定时停止容器

API 接口

# 方式一：Header
curl -H "x-request-api-key: YOUR_API_KEY" http://localhost:8000/api/containers

# 方式二：Query 参数
curl http://localhost:8000/api/containers?apikey=YOUR_API_KEY

环境变量配置

以下环境变量可在启动时设置，用于覆盖默认行为：

变量名默认值说明
PORT8000服务监听端口
CORS_ORIGINSlocalhost允许的 CORS 源列表（逗号分隔）
COOKIE_SECUREfalseCookie 的 Secure 标志，HTTPS 环境设为 true

示例

# Linux / macOS
export CORS_ORIGINS="https://mysite.com,https://admin.mysite.com"
export COOKIE_SECURE=false

$env:CORS_ORIGINS = "https://mysite.com"

python start.py --port 9000
生产环境部署时，务必设置 CORS_ORIGINS 为实际域名，避免使用通配符。
项目提供了 .env.example 文件，可复制为 .env 后按需修改。

安全说明

NapCat Manager 内置了多层安全防护机制，以下是各项安全特性说明：

认证与授权

特性说明
双模认证支持 Cookie Session 和 API Key 两种认证方式
密码加密使用 bcrypt 哈希存储密码，支持 SHA256 旧格式自动迁移
密码策略密码最低 6 个字符
随机初始密码应急管理员使用 secrets.token_urlsafe(12) 生成随机密码，仅在终端输出一次
IP 封禁连续登录失败后自动封禁 IP 一段时间
会话管理定期清理过期会话令牌

请求防护

特性说明
CSRF 防护对 Cookie 认证的写操作校验 X-Requested-With: XMLHttpRequest 头（API Key 认证和 GET 请求自动豁免）
SSRF 防护Webhook URL 自动解析 DNS 并拒绝内网/回环/保留地址（可通过告警设置页面的「允许所有 IP 访问」开关关闭此限制）
路径遍历防护文件管理使用 os.path.realpath() 校验，防止 ../ 攻击
容器名称校验仅允许字母、数字、连字符、下划线和点号，防止命令注入
备份上传限制最大 50 MB，双重校验（Content-Length + 实际大小）
全局异常处理兜底异常处理器避免向客户端暴露内部堆栈信息
频率限制创建容器等敏感操作有频率限制

用户面板（公开端点）

以下端点无需认证，专为外部用户面板设计：

  • GET /api/public/containers — 返回容器名称、状态、QQ 号（uin），不暴露端口等敏感信息
  • GET /api/containers/{name}/qrcode — 返回 QQ 登录二维码
这两个端点是公开的，请确保不要在其中暴露敏感数据。如果您的管理面板部署在公网，建议使用反向代理限制访问源。

Docker 部署详解

使用 Docker Compose

# docker-compose.yml 配置概览
services:
  ncqq-manager:
    build: .
    container_name: ncqq-manager
    ports:
      - "8000:8000"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock  # 必须：管理宿主机容器
      - ./config:/app/config                        # 持久化配置
      - ./data:/app/data                            # 持久化数据
    environment:
      - CORS_ORIGINS=                               # 可选：CORS 允许源
      - COOKIE_SECURE=false                         # 可选：HTTPS 时设为 true
    restart: unless-stopped

使用 docker run

# 先构建镜像
docker build -t ncqq-manager .

# 运行容器
docker run -d \
  --name ncqq-manager \
  -p 8000:8000 \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v $(pwd)/config:/app/config \
  -v $(pwd)/data:/app/data \
  --restart unless-stopped \
  ncqq-manager

健康检查

Docker 镜像内置了 HEALTHCHECK 指令，每 30 秒检查一次 /api/health 端点。可通过以下命令查看健康状态：

docker inspect --format='{{.State.Health.Status}}' ncqq-manager
Docker 部署时，容器通过挂载 /var/run/docker.sock 来管理宿主机上的其他 Docker 容器。确保 Docker socket 文件权限正确。

常见问题

Q: 启动后页面显示 "Frontend not built yet"？

A: 前端尚未构建。执行以下命令：

cd frontend && npm install && npm run build

或直接使用快速启动脚本 python start.py 自动构建。

Q: 容器管理页面提示 Docker 连接失败？

A: 检查以下几点：

  • Docker 服务是否正在运行：docker info
  • 当前用户是否有 Docker 权限（Linux 需加入 docker 组）
  • Docker Desktop（Windows/macOS）是否已启动

Q: 如何重置管理员密码？

A: 有两种方式：

  1. 推荐：停止服务 → 删除 config/app.db → 重启服务 → 系统自动生成随机密码的应急管理员（密码会在终端日志中输出一次）→ 登录后立即修改密码
  2. 备选：如果有其他管理员账号，可在用户管理页面重置密码

Q: QR 码一直显示"等待生成"？

A: 可能原因：

  • 容器未运行 — 先启动容器
  • NapCat 尚在初始化 — 等待 10-20 秒
  • WebUI 端口配置错误 — 检查 config/webui.json 中的端口设置

Q: 如何添加远程节点？

A: 参见 集群配置 章节。确保远程节点的 api_key 配置正确。

Q: 支持 HTTPS 吗？

A: 推荐使用 Nginx/Caddy 反向代理配置 HTTPS：

# Nginx 示例（含 WebSocket 支持）
server {
    listen 443 ssl;
    server_name your-domain.com;
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket 支持（容器状态推送 + 日志流）
    location /ws/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
使用反向代理时，记得将 CORS_ORIGINS 设置为你的域名，并将 COOKIE_SECURE 设为 true。

Q: 数据保存在哪里？

A: 所有数据位于项目根目录下：

目录/文件内容
config/config.json启动引导配置（initialized / host / port）
config/app.dbSQLite 数据库（用户、会话、节点、运行时设置、操作日志）
data/{容器名}/各容器挂载数据
